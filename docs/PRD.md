# PRD — 팀 프로젝트 일정관리 툴

## 1. 제품 개요

Jira/GitHub Projects에 익숙하지 않은 소규모 팀(3명)을 위한 단순하고 직관적인 프로젝트 일정관리 **웹 서비스**.
칸반보드로 태스크를 시각화하고, WBS로 프로젝트 진행 상황을 계층적으로 모니터링한다.
1000school(GCS-PULSE)과 연동해 팀 멤버 조회, 회의실 예약 관리를 처리하고, Google Calendar와 연동해 예약 이벤트를 자동 생성·삭제한다.

---

## 2. 사용자

- 팀 구성: 소규모 팀 (1000school 플랫폼에 등록된 팀원)
- 기술 숙련도: Jira/GitHub Projects 미숙
- 목표: 최소한의 학습으로 프로젝트 진행 상황 파악 및 협업

---

## 3. 화면(뷰) 구성

| 뷰 | 설명 |
|----|------|
| **보드** | 칸반 방식으로 태스크를 관리 (기본 뷰) |
| **WBS** | 에픽 → 태스크 → 서브태스크 계층 구조 |
| **타임라인** | 프로젝트 일정 Gantt 차트 뷰 |
| **문서** | 프로젝트 관련 문서 관리 |
| **회의실** | 회의실 예약 타임테이블 및 내 예약 목록 |
| **설정** | 팀 멤버 관리, Google Calendar 연동, 1000school API 토큰 설정 |

---

## 4. 핵심 기능

### 4.1 칸반보드

- 태스크를 상태(컬럼)별로 시각화
- 컬럼: `Todo` / `In Progress` / `In Review` / `Done`
- 태스크 카드: 제목, 담당자 아바타, 우선순위 배지, D-day 마감일 표시 (빨강/주황/회색)
- 드래그 앤 드롭으로 상태 변경
- 컬럼 영역 클릭으로 해당 상태로 새 태스크 즉시 생성
- **보드 필터**: 텍스트 검색, 담당자, 우선순위, 태그 필터 동시 적용
- 프로젝트별 보드 분리

### 4.2 태스크 상세

- 제목, 설명, 담당자, 우선순위, 마감일, 태그 수정 → DB 즉시 저장
- **담당자**: 팀 멤버 목록에서 선택 (팀에서 가장 많이 지정된 멤버 상단 노출)
- **태그**: 입력 자동완성 (프로젝트 내 기존 태그 제안)
- **서브태스크(체크리스트)**: DB 저장 + 완료 퍼센트 표시
- **댓글**: 작성/삭제 DB 저장

### 4.3 WBS (Work Breakdown Structure)

- 3단계 계층 구조: **에픽(Level 1) → 태스크(Level 2) → 서브태스크(Level 3)**
- 에픽: 프로젝트의 대단위 목표
- 태스크: 칸반보드 카드와 동일한 엔티티 (WBS와 칸반보드가 같은 태스크를 공유)
- 서브태스크: 태스크 하위의 세부 작업 (완료 체크박스)
- 진행률 자동 계산 (완료된 서브태스크 / 전체 서브태스크)

### 4.4 회의실 예약

- 1000school에 등록된 회의실 목록 조회
- 타임테이블 UI: 9시~24시 전체 슬롯 가로 스크롤 가능
- 빈 슬롯 클릭 → 해당 시간 기준 1시간 예약 생성 모달 즉시 표시
- 예약 완료 시 참석자 이메일로 Google Calendar 이벤트 자동 생성 + 캘린더 초대 발송
- 예약 취소 시 Google Calendar 이벤트 동시 삭제
- **내 예약** 탭: 오늘 이후 본인 예약 목록 조회 + 바로 취소

### 4.5 팀 멤버 관리 (설정)

- 1000school API 토큰 입력 → 팀 멤버 자동 동기화
- 프로젝트별 멤버 역할(admin/member) 관리
- 담당자 지정 시 이 멤버 목록에서 조회

---

## 5. 범위 외 (Out of Scope)

- 팀/멤버 생성·삭제 (1000school에서 관리, 앱은 조회·동기화만)
- 실시간 알림 push (SSE relay 미구현)
- 모바일 앱
- 권한별 기능 제한 (현재 role은 저장만, UI 제한 없음)

---

## 6. 1000school(GCS-PULSE) 연동 요약

| 기능 | 1000school 엔드포인트 | 방식 |
|------|----------------------|------|
| 팀/멤버 조회 | `GET /teams/me` | 사용자 API 토큰 |
| 회의실 목록 | `GET /meeting-rooms` | 사용자 API 토큰 |
| 날짜별 예약 현황 | `GET /meeting-rooms/{id}/reservations` | 사용자 API 토큰 |
| 회의실 예약 생성 | `POST /meeting-rooms/{id}/reservations` | 사용자 API 토큰 |
| 예약 취소 | `DELETE /meeting-rooms/reservations/{id}` | 사용자 API 토큰 |
| 내 예약 목록 | `GET /meeting-rooms/reservations/my` | 사용자 API 토큰 |

사용자 API 토큰은 설정 페이지에서 입력하며, `users.school_api_token`에 저장된다.

---

## 7. 비기능 요구사항

- 배포: Railway (백엔드 + PostgreSQL) / Vercel (프론트엔드)
- 응답 속도: 일반 CRUD 200ms 이내
- 동시 사용자: 최대 50명 (PostgreSQL + Railway 적합 범위)
- API 문서: Swagger UI (`http://localhost:8000/docs`) 자동 제공
